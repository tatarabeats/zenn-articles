## TL;DR

- SBI証券と銀行アプリを行き来するのが嫌で自分で作った
- コードは**1行も書いていない**。全部Claude Codeに任せた
- 期間は**約1週間**
- コスト: Claude Max 3万円（もっと安いプランでも作れた）

---

## なぜ作ったか

**毎朝2つのアプリを開くのが嫌だった。**

SBI証券と住信SBIネット銀行。この2つを行き来しないと総資産が分からない。しかも投資信託と個別株を両方持っていると、SBI証券の保有資産欄がちゃんと表示されないことがある。

「今、自分は全部でいくら持ってるんだ？」という一番シンプルな疑問に、すぐ答えられない。

マネーフォワードも試した。でも無料版はリアルタイム反映されないし、有料版は月額課金が発生する。UIもダサい。資産管理のためだけにサブスクを増やすのが嫌だった。

だったら自分で作ればいい。そのときClaude Codeがあることを思い出した。

![ダッシュボード](/images/demo-1-home.gif)

## どうやって作ったか

Claude Codeに指示を出した。それだけ。

技術スタックはClaudeが決めた：

- **Next.js + TypeScript**（フロント）
- **Turso（クラウドSQLite）+ Drizzle ORM**（DB）
- **Tailwind CSS v4 + shadcn/ui**（デザイン）
- **Vercel**（デプロイ）
- **Auth.js v5 + Google OAuth**（認証）

自分がやったのは「このカードの色を変えて」「アニメーションを滑らかにして」という指示だけ。機能実装のコストが低かった分、開発期間の大半を**見た目と使い心地の改良に使えた**。

---

## Claude Codeを使って分かったこと

一番驚いたのは、**作業中にYouTubeを見ていられること**だ。

指示を出したら、あとはClaudeが勝手にやってくれる。ファイルの整理も、実装の方針決定も、エラーの修正も。できあがったものをエクスプローラーで確認するだけでいい。

「生産するためには集中して作業しなければならない」という思い込みがあった。でもClaude Codeを使うと、その呪いが解ける。

**消費者と同じ感覚で生産できる。**

「こういうのが欲しい」と言えば出てくる。ダラダラしながら、アプリが育っていく。
